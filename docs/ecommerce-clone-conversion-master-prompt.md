# Ecommerce Clone Conversion Master Prompt

이 문서는 타겟 이커머스 사이트를 구조 검증용 클론으로 만들 때 반복 사용할 마스터 프롬프트다.

목표는 새 브랜드 전략을 만드는 것이 아니라, 원본 사이트의 전환 구조와 구매 흐름은 최대한 유지하면서 원본 고유 표현물만 안전하게 치환하는 것이다.

## 사용 입력

작업 시작 시 아래 값을 채운다.

```text
타겟 원본 사이트 URL:
작업 브랜드명:
테스트 상품 목록:
우선 파일럿 상품:
로컬 클론 사이트 URL:
상품 이미지 생성 도구:
최종 산출물 형식:
```

## Codex에게 입력할 전체 프롬프트

```text
너는 이커머스 클론 사이트의 전환 구조 유지 및 고유 표현물 치환을 총괄하는 작업자다.

목표:
원본 사이트의 돈 버는 구조, 구매 흐름, 상품 노출 방식, 상세페이지 흐름은 최대한 유지한다.
하지만 원본의 브랜드명, 로고, 상품명, 문구, 이미지, 후기, 상세페이지 속 제품 컷은 그대로 남기지 않는다.

중요:
이 작업은 새로운 사이트 디자인이 아니다.
타겟 사이트의 전환 구조를 검증하기 위한 클론 사이트 완성 작업이다.
따라서 구조는 원본과 거의 같게 유지하고, 표현물만 우리 것으로 바꾼다.

브랜드명:
{{작업 브랜드명}}

타겟 원본 사이트:
{{타겟 원본 사이트 URL}}

이번에 작업할 테스트 상품:
{{테스트 상품 목록}}

파일럿 상품:
{{우선 파일럿 상품}}

작업 방식:
1. 먼저 원본 사이트와 현재 로컬 클론 사이트의 구조를 파악한다.
2. 상품은 사용자가 지정한 테스트 상품만 진행한다.
3. 먼저 파일럿 상품 1개만 끝까지 진행한다.
4. 사용자가 OK하면 나머지 상품으로 확장한다.

전체 사이트에서 유지할 것:
- 메인페이지 흐름
- 카테고리 구조
- 상품 노출 순서
- 상품 리스트 구조
- 상세페이지 섹션 순서
- 구매 버튼 위치
- 옵션 선택 위치
- 장바구니/구매 흐름
- 후기 배치 방식
- 배송/교환/보증/FAQ 위치
- 전환 유도 흐름

전체 사이트에서 바꿀 것:
- 원본 브랜드명
- 원본 로고
- 원본 상품명
- 원본 문장
- 원본 후기
- 원본 고객명/프로필
- 원본 이미지
- 원본 상세페이지 이미지 속 제품 컷
- 원본 이미지 속 문구/로고
- alt 텍스트
- 메타 문구
- 파일명 안의 원본 브랜드명

상세페이지 처리 원칙:
상세페이지가 이미지로 되어 있으면, 그 이미지를 그대로 쓰지 않는다.
분절 이미지 한 장 안에 텍스트와 사진이 섞여 있으면 반드시 둘을 분리한다.

분절 이미지 처리 기준:
1. 원본 분절 이미지 전체를 Atlas/ChatGPT에 넣지 않는다.
2. 분절 이미지 안에서 실제 사진 요소만 crop한다.
3. 텍스트 영역은 이미지로 쓰지 않고 HTML 텍스트로 재작성한다.
4. 아이콘/표/FAQ/사이즈/제품 정보는 가능하면 HTML/CSS로 재구성한다.
5. crop 단위는 실제 이미지 생성에 넣을 원본 사진 단위다.

예:
잘못된 입력 단위:
- 11-product-only.jpg 전체

올바른 입력 단위:
- 11-01-wide-folded-fabric.jpg
- 11-02-outside-fabric-texture.jpg
- 11-03-inside-fabric-texture.jpg

이미지 crop 기준:
- 제품 단독컷
- 사람 착용컷
- 원단 확대컷
- 물 흡수 데모컷
- 손으로 늘리는 기능 데모컷
- 색상 옵션 제품컷
- 브랜드 무드컷

crop 제외 기준:
- 텍스트만 있는 이미지
- 아이콘만 있는 이미지
- FAQ 이미지
- 사이즈 도식 이미지
- 제품 정보표 이미지
- 배송/교환/세탁 안내 이미지
- 인증서 이미지

단, crop 제외 대상도 상세페이지에서는 버리지 않는다.
원본 이미지로 쓰지 않고 HTML 텍스트, HTML 표, CSS 레이아웃, 아이콘형 블록으로 재작성한다.

GIF/영상형 이미지 처리:
GIF는 첫 프레임을 무조건 쓰지 않는다.
의미 있는 순간을 프레임별로 확인한다.

예:
흡수력 데모 GIF라면 아래 장면을 골라 crop한다.
- 물방울이 원단에 닿는 순간
- 물이 원단 위에 고이는 순간
- 물이 흡수되어 젖은 자국이 남는 순간

의미 없는 프레임:
- 물이 아직 떨어지지 않은 상태
- 병만 보이고 기능이 드러나지 않는 상태
- 기능 결과가 확인되지 않는 상태

중복/유사 이미지 처리:
같은 상품 상세페이지 안에서 거의 같은 사진이 반복될 수 있다.
예를 들어 대표 이미지, 일반 착용컷, 기능 설명용 착용컷이 같은 인물/같은 각도/같은 착용 상태를 반복할 수 있다.
이 경우 각 이미지를 무조건 별도 생성 대상으로 보지 않는다.

반드시 아래를 판단한다.
- 같은 사진을 거의 그대로 반복하는가?
- crop 후에도 인물, 각도, 착용 위치, 배경이 거의 같은가?
- 상세페이지에서 역할만 다른가?
- 하나의 생성 결과를 재사용해도 되는가?
- 기능 섹션이라 별도 변형이 필요한가?

엑셀에는 중복/유사 이미지 그룹을 별도 열에 기록한다.

예:
- main.png
- 05-01-white-wearing.jpg
- 12-01-uv-white-wearing.jpg

위 세 장이 거의 같은 흰색 헤드밴드 착용컷이라면 같은 유사 그룹으로 묶는다.
대표 이미지는 썸네일용으로 우선 생성하고, 상세페이지 안에서는 같은 결과를 재사용하거나 필요한 경우 배경/크롭/문구 없는 버전만 변형한다.

특히 아래 같은 경우는 중복 생성 대상이 아니다.

```text
main.png
05-01-white-wearing.jpg
12-01-uv-white-wearing.jpg
```

위처럼 인물, 얼굴 각도, 착용한 상품, 배경 톤, 촬영 무드가 거의 같으면 `같은 원본 착용컷 반복`으로 판단한다.
이때는 세 장을 각각 Atlas에 넣어 새로 만들지 말고, 먼저 대표 기준 이미지를 하나 정한다.
나머지는 아래 중 하나로 처리한다.

- 같은 결과 이미지를 재사용
- 상세페이지 위치에 맞춰 크롭만 변경
- 기능 섹션용이면 배경/색감/구도만 약하게 변형
- 별도 기능 의미가 없으면 생성 대상에서 제외

엑셀 처리:
- 같은 그룹명 예: `WHITE-WEARING-A`
- 대표 이미지 처리 메모: `대표/썸네일 기준 이미지`
- 중복 이미지 처리 메모: `대표 결과 재사용 또는 크롭/배경 변형`

중복/유사 이미지 처리 원칙:
1. 같은 사진을 여러 번 Atlas에 넣어 중복 생성하지 않는다.
2. 먼저 하나의 기준 이미지를 정한다.
3. 같은 그룹의 나머지는 재사용/변형/생략 중 하나로 판단한다.
4. 기능 의미가 다른 경우에만 별도 프롬프트를 작성한다.
5. 엑셀에 `유사 이미지 그룹`과 `처리 메모`를 기록한다.
6. 같은 착용컷이 대표 이미지와 상세페이지 중간에 반복되면, 상세페이지 구조상 반복 위치는 유지하되 생성 작업은 중복하지 않는다.

이미지 생성 배치 운영:
crop된 원본사진 수와 실제 생성 작업 수는 다를 수 있다.
먼저 중복/유사 이미지 그룹을 정리한 뒤, 실제 Atlas/ChatGPT에 넣을 생성 작업 수를 확정한다.

예:
- crop 원본사진 19개
- `main.png / 05-01-white-wearing.jpg / 12-01-uv-white-wearing.jpg`는 같은 흰색 착용컷 그룹
- `17-02-wide-width-product.jpg / 19-01-black-color-product.jpg`는 같은 블랙 제품컷 그룹
- 따라서 실제 생성 작업은 16개

배치 방식:
1. 실제 생성 작업 목록을 만든다.
2. 한 창에는 원본 이미지 1개와 해당 프롬프트 1개만 넣는다.
3. 여러 이미지를 한 창에 몰아서 넣지 않는다.
4. 여러 프롬프트를 한 창에 섞지 않는다.
5. ChatGPT/Atlas 창을 8개 열어 1차 배치를 돌린다.
6. 1차 8개 입력 후 약 3분 기다린다.
7. 다시 ChatGPT/Atlas 창을 8개 열어 2차 배치를 돌린다.
8. 각 창에서 받은 결과 이미지를 저장한다.
9. 엑셀의 `받아낸 이미지 파일명` 열에 결과 파일명을 기록한다.
10. 중복 그룹에 속한 이미지는 같은 생성 결과를 재사용하거나 크롭/배경 변형만 적용한다.

생성 배치표에는 아래 열을 둔다.
- 배치 번호
- 작업 순서
- 생성 여부
- 기준 원본사진 파일명
- 재사용 대상 파일명
- Atlas 입력 프롬프트
- 결과 이미지 파일명
- 메모

이미지 생성 도구 사용 원칙:
이미지 변환은 Atlas 안의 ChatGPT 웹을 사용한다.
Codex 내부 이미지 생성 도구를 사용했다면 반드시 그렇게 밝힌다.
실제로 Atlas/ChatGPT에 원본 이미지를 넣지 않았다면 넣었다고 말하지 않는다.

Atlas/ChatGPT 입력 방식:
1. crop된 원본 사진을 넣는다.
2. 해당 사진의 역할에 맞는 프롬프트를 같이 넣는다.
3. 원본의 기능적 의미는 유지한다.
4. 원본 인물, 배경, 조명, 구도, 접힘, 확대 비율은 다르게 만든다.
5. 로고/브랜드명/원본 문구는 제거한다.

사람 착용컷 프롬프트 기본 방향:
원본 착용컷을 참고해서 얼굴이 좌측을 향한 한국 남성 광고 모델 느낌으로 바꾸고, 배경은 차분한 파란색 스튜디오 배경으로 변경. 착용한 상품의 위치와 기능적 의미는 유지하되 원본 인물, 원본 배경, 원본 조명, 원본 얼굴 형태와 다르게 생성. 실제 착용컷처럼 자연스럽게. 로고와 브랜드명 없음.

제품/원단컷 프롬프트 기본 방향:
원본 제품 사진을 참고해서 같은 상품 유형의 제품/소재 컷을 새로 만들어줘. 기능적 의미와 제품 종류는 유지하되 원본과 같은 배경, 조명, 구도, 접힘, 확대 비율처럼 보이지 않게 바꿔줘. 로고와 브랜드명은 제거.

작업 산출물:
1. crop된 실제 원본사진 폴더
2. 상세페이지 스크립트 문서
3. Atlas Prompt Plan 엑셀
4. HTML Only 재작성 대상 목록
5. 최종 검수 체크리스트

엑셀 구조:
Detail Script 시트:
- 순서
- 섹션 제목
- 이미지 표기
- 텍스트
- 처리 방식

이미지 표기 형식:
(이미지, crop된 원본사진 파일명)

Atlas Prompt Plan 시트:
- A열: 삽입 위치
- B열: 실제 crop 원본사진 썸네일
- C열: crop 원본사진 파일명
- D열: 원본 분절 파일
- E열: 이미지 역할
- F열: 상세페이지 위치
- G열: Atlas 입력 프롬프트
- H열: 받아낸 이미지 파일명
- I열: 유사 이미지 그룹
- J열: 처리 메모

HTML Only 시트:
- 원본 분절 파일
- 역할
- 처리 방식
- 비고

Checklist 시트:
- crop 원본사진 확인
- Atlas 입력 확인
- 결과 이미지 파일명 기록
- 사이트 삽입
- 텍스트 HTML 재작성
- 원본 브랜드/로고/문구 제거
- 모바일 검수
- 구매 흐름 검수

작업 순서:
1. 타겟 원본 사이트 전체 구조 파악
2. 현재 로컬 클론 사이트 구조 파악
3. 테스트 상품 목록 확정
4. 파일럿 상품 1개 선택
5. 파일럿 상품 상세페이지 원본 이미지/HTML 수집
6. 상세페이지가 HTML형/분절 이미지형/긴 통이미지형/혼합형인지 판별
7. 분절 이미지 전체 목록 생성
8. 각 분절 이미지의 역할 판단
9. 분절 이미지 안에서 실제 사진 요소만 crop
10. 텍스트/아이콘/FAQ/표/사이즈/제품정보는 HTML Only 대상으로 분리
11. GIF는 프레임별로 확인하고 기능 의미가 드러나는 프레임만 추출
12. crop된 사진끼리 중복/유사 여부를 비교
13. 대표 생성 이미지와 재사용/변형 대상을 구분
14. crop된 실제 사진 기준으로 Atlas Prompt Plan 작성
15. 상세페이지 텍스트를 Runnerwin/작업 브랜드 기준으로 재작성
16. 엑셀에 실제 crop 사진 썸네일, 파일명, 원본 분절 파일, 프롬프트, 유사 이미지 그룹을 정리
17. 사용자가 crop/프롬프트를 확인
18. Atlas/ChatGPT로 이미지를 생성

## 이번 작업에서 확정된 핵심 교훈

1. `원본사진 이름`은 원본 분절 이미지 파일명이 아니라 Atlas에 넣을 실제 crop 사진 파일명이어야 한다.
2. `B열 원본사진`에는 crop된 실제 사진 썸네일이 들어가야 한다.
3. `D열 원본 분절 파일`로 어느 상세페이지 조각에서 잘랐는지 추적한다.
4. 분절 이미지 안의 문구는 이미지 생성 대상이 아니라 HTML 재작성 대상이다.
5. GIF는 첫 프레임을 쓰면 안 된다. 기능 의미가 보이는 프레임을 골라야 한다.
6. 제품이 잘린 crop은 실패다. 제품 전체 또는 기능 장면이 명확히 보여야 한다.
7. 사람 착용컷은 사람 착용컷이라고 명확히 분류한다.
8. 제품컷, 원단컷, 데모컷, 착용컷을 섞어서 같은 프롬프트로 처리하지 않는다.
9. 거의 같은 착용컷은 반드시 유사 이미지 그룹으로 묶고, 중복 생성하지 않도록 재사용/변형 기준을 적는다.

# 가격 변경 광고소재 동기화 검토 기준

잇템몰 상품 가격을 바꾸고 Meta 광고를 게시할 때는 가격 숫자가 들어가는 모든 곳을 한 번에 맞춘다.

## 기본 원칙

- 사용자가 판매가를 직접 정하면 그 값을 최종 판매가로 보고, 기존 네이버 쇼핑 가격 산정값보다 우선한다.
- 사이트 상품가, 서버 상품 카탈로그, JSON-LD 구조화 데이터, QA 스크립트, 광고 문구, 광고 이미지, 광고 패키지 기록을 함께 확인한다.
- 광고 이미지 안에 가격이 적혀 있으면, 이미지 가격이 새 판매가와 같아지기 전에는 게시하지 않는다.
- 예전 가격이 들어간 광고는 그대로 켜지 않는다. 새 가격 이미지로 새 광고 소재와 새 광고를 만들거나, 가격이 맞는 소재로 교체한 뒤 게시한다.

## 가격 변경 체크

1. `index.html` 상품 데이터의 `price`와 JSON-LD `offers.price`를 새 가격으로 맞춘다.
2. `payment/order-store-lib.php`의 서버 상품 가격을 새 가격으로 맞춘다.
3. 추가상품이 있으면 기본가 + 추가상품 총액을 다시 계산한다.
4. N pay와 바로구매 로그 payload의 `value`, `addOnTotal`, `selectedOptions`가 새 총액을 보낼 수 있는지 확인한다.
5. QA 스크립트의 기대 가격 문구를 새 가격으로 바꾼다.
6. Meta 광고 문구와 랜딩 URL UTM 이름에 예전 가격/구성이 남아 있는지 확인한다.
7. 가격이 들어간 광고 이미지 파일을 새 가격으로 교체하고 눈으로 확인한다.

## 예산 계산

- 기본 허용 광고비는 `판매가 x 20%`로 계산한다.
- 목표 CPC는 `판매가 x 0.004`로 계산한다.
- 사용자가 별도 지출 상한을 주면, Meta 광고 예산과 별개로 `지출 상한 도달 시 중지` 자동 점검을 설정한다.

## 게시 전 확인

- 사이트 가격과 광고 이미지 가격이 같아야 한다.
- 상품 기본 구성과 추가상품 설명이 사이트 본문과 광고 문구에서 서로 맞아야 한다.
- 새 가격 광고는 기존 가격 광고세트/광고 성과와 섞이지 않도록 새 광고세트 또는 새 광고 단위로 분리한다.
- 게시 후 자동 점검이 필요한 경우 광고비 점검 주기, 로그 점검 주기, 중지 기준, 대상 광고 ID를 기록한다.

## 기록

게시 기록에는 아래를 남긴다.

- 판매가
- 추가상품 가격과 선택 총액
- 허용 광고비와 목표 CPC
- 광고 이미지 파일
- Meta campaign/adset/ad/creative ID
- 게시 시작 KST 시각
- 광고비 중지 기준
- 구매의사 로그 집계 기준

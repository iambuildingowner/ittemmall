# 잇템몰 아이템 테스트 지침

## 워크프로세스 Mermaid 기준

- 사용자가 이 프로젝트에서 `워크프로세스 머메이드`, `워크플로우 머메이드`, `프로세스 도표`, `현재 프로젝트 워크플로우`를 물으면 `잇템몰_아이템_테스트_개요.md` 안의 Mermaid 2개를 기준으로 답한다.
- 첫 번째 기준 도표는 `현재 기준 워크플로우 (2026-07-03)`이다. 잇템몰 아이템 테스트 전체 흐름으로, 사용자 입력부터 아이템 발굴, 상품 업로드, 상세페이지 제작, N pay 버튼 픽셀/서버/메일 검증, Meta 광고 검증, 성과 판단까지 이어지는 도표다.
- 두 번째 기준 도표는 `기술 워크플로우: 상품별 픽셀/서버 로그 필수 (2026-07-02)`이다. 상품 상세의 일반 구매/N pay 버튼 클릭부터 상품별 이벤트명, localStorage, dataLayer, Meta Pixel, `track.php`, 서버 로그, 메일 알림, 테스트/실제 성과 판단까지 이어지는 도표다.
- 광고 검증이나 광고 감시를 시작하기 전에는 `08_monitoring/광고_검증_착수전_점검_기준.md`를 먼저 본다. 이 문서의 핵심은 `조회 불가를 0건으로 말하지 않기`, `Meta 픽셀/서버 로그/이메일을 따로 표기하기`, `중복 제거 기준을 출처별로 구분하기`다.
- 별도 지시가 없으면 `신사업발굴` 폴더의 CaseFlow, Alibaba 견적 수집 워크플로우, 다른 프로젝트의 상태머신을 이 프로젝트의 기본 워크프로세스 Mermaid로 말하지 않는다.
- 답변할 때는 `잇템몰_아이템_테스트_개요.md`의 두 도표가 기준이라고 먼저 말하고, 필요한 경우 어떤 도표를 봐야 하는지 구분해서 안내한다.

## 팔찌/조끼 수익표 기본값

- 사용자가 팔찌, 조끼, 원가표, 투자대비수익률, 수익률을 묻고 `알리바바`, `Alibaba`, `중국소싱`, `중국 원가`, `알리바바 가격`을 명시하지 않으면 알리바바 견적표나 알리바바 원가를 기본으로 쓰지 않는다.
- 기본 답변은 국내소싱 기준 저장 문서를 먼저 본다.
  - `08_monitoring/2026-07-07_국내소싱_도매가_기준_손익_재계산.md`
  - `08_monitoring/2026-07-07_중국소싱_대비_국내소싱_원가_수익률_비교.md`
- 즉 `조끼랑 팔찌 원가표`, `투자대비수익률 표`처럼만 말하면 국내소싱 기준 요약을 먼저 제시한다.
- 국내소싱 기준 핵심값은 팔찌 투자비 대비 수익률 `52.2%`, 조끼 투자비 대비 수익률 `29.3%`다.
- 알리바바 가격은 사용자가 알리바바, 중국소싱, 중국 원가, 알리바바 가격 비교를 명시했을 때만 함께 보여준다.

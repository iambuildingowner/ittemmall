# 잇템몰 광고 검증 착수 전 점검 기준

잇템몰 상품 광고를 새로 게시하거나, 이미 게시된 광고의 반응을 읽기 전에는 아래 점검을 먼저 한다.

## 먼저 확인할 것

| 구분 | 확인 내용 | 실패하면 |
|---|---|---|
| Meta 광고비 | 광고 ID 기준 지출, 노출, 클릭, 랜딩페이지 조회가 조회되는가 | 광고비 숫자를 UNKNOWN으로 둔다 |
| Meta 고유 클릭 | `unique_clicks`, `unique_inline_link_clicks`가 조회되는가 | 고유 클릭은 미확정으로 둔다 |
| Meta 픽셀 | 상품별 `NpayPurchaseClick_{상품slug}`, `PurchaseCtaClick_{상품slug}`가 조회되는가 | 픽셀 구매의사 수를 미확정으로 둔다 |
| 서버 저장 | 테스트 ID로 상품별 버튼 이벤트가 `stored:true`가 되는가 | 광고 게시/성과 판단을 멈춘다 |
| 서버 중복 제거 | 테스트 ID로 같은 방문자 N pay 2회 + 다른 방문자 바로구매 1회를 남긴 뒤 `총 3클릭 -> 중복 제거 구매의도 2건`이 되는가 | 서로 다른 구매의도자인지 판단하지 않는다 |
| 서버 로그 조회 | 본사이트 서버 로그 또는 `payment/tracking-report.php`가 읽히는가 | 서버 로그 수를 0건으로 말하지 않는다 |
| 이메일 알림 | 테스트 ID가 붙은 메일 알림이 발송/수신 확인되는가 | 이메일 알림 수를 미확정으로 둔다 |

## 숫자 보고 원칙

- `조회 불가`와 `0건`은 다르다.
- 서버 로그 경로가 `MISS:0`이면 `0건`이 아니라 `로그 경로 미확정`이다.
- Gmail 권한이 없어서 메일을 못 읽으면 `메일 확인 불가`라고 말한다.
- 운영 서버의 `notification.enabled`가 `false`이면 메일 알림 0건이 아니라 `알림 설정 꺼짐`으로 말한다.
- Meta 픽셀 이벤트 수가 있더라도 서버 로그 중복 제거 방문자 수로 바꿔 말하지 않는다.
- Meta 픽셀 구매의사 3건은 `클릭 3회`다. 서로 다른 구매의도자 3명인지 보려면 서버 로그의 중복 제거 결과가 있어야 한다.

## 중복 제거 기준

Meta 광고 쪽:

- 광고 클릭은 `unique_clicks`, 링크 클릭은 `unique_inline_link_clicks`를 함께 본다.
- Meta 픽셀 stats의 custom event count는 이벤트 수다. 방문자 중복 제거 수로 단정하지 않는다.

서버 로그 쪽:

1. `visitorId`
2. `visitId`
3. `clickId`
4. `eventId` 또는 `pixelEventId`
5. `serverRecordId`
6. `ipHash`

위 순서로 중복 제거한다.

## 기본 보고표

| 구분 | 값 | 출처 | 상태 |
|---|---:|---|---|
| 광고비 |  | Meta Ads Insights | 조회/조회불가 |
| 전체 클릭 |  | Meta Ads Insights `clicks` | 조회/조회불가 |
| 링크 클릭 |  | Meta Ads Insights `inline_link_clicks` | 조회/조회불가 |
| 고유 링크 클릭 |  | Meta Ads Insights `unique_inline_link_clicks` | 조회/조회불가 |
| 랜딩페이지 조회 |  | Meta Ads Insights `landing_page_view` | 조회/조회불가 |
| N pay 구매의사 |  | Meta Pixel `NpayPurchaseClick_{상품slug}` | 조회/조회불가 |
| 바로구매 구매의사 |  | Meta Pixel `PurchaseCtaClick_{상품slug}` | 조회/조회불가 |
| 서버 로그 구매의사 |  | 본사이트 서버 로그 | 조회/조회불가 |
| 이메일 알림 |  | Gmail 또는 메일 수신 확인 | 조회/조회불가 |

## 자동 점검 도구

가능하면 아래 스크립트로 먼저 점검한다.

```bash
python3 scripts/check_ittemmall_ad_tracking_preflight.py \
  --product-slug windcool-vest \
  --product-id product-003 \
  --ad-id 120248598186740703 \
  --adset-id 120248598179870703 \
  --campaign-id 120248369875410703 \
  --pixel-id 1288524852852406 \
  --since-utc 2026-07-07T19:09:17+0000 \
  --until-utc 2026-07-07T22:45:22+0000
```

`--server-click-test`를 붙이면 테스트 ID로 서버 저장/삭제까지 확인한다.
이 테스트는 같은 방문자의 N pay 2회와 다른 방문자의 바로구매 1회를 남긴 뒤 삭제한다. 삭제 응답의 `dedupe_proof.passed`가 `true`여야 광고 착수 전 로그 점검 통과로 본다.
`--notify-test`는 테스트 메일 발송까지 시도하므로, 사용자가 메일 테스트를 허용했을 때만 붙인다.

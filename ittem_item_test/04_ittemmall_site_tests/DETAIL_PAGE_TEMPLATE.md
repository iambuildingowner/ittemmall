# 잇템몰 상세페이지 기본 템플릿

목적: 잇템몰 상품 업로드 시 기존 마르세 상세페이지 구조를 재사용하되, 상품별 이미지/카피/포인트/사양만 바꿔 빠르게 상세페이지를 만든다.

## 기본 원칙

- `상품 업로드`에는 상세페이지 제작과 N pay 버튼 픽셀 세팅을 기본 포함한다.
- N pay 버튼 클릭은 실제 네이버페이 결제가 아니라 구매 의향 반응으로 본다.
- Toss 결제수단은 차후 실제 Toss 결제를 받을 때만 별도로 다룬다.
- 배송, 교환, 환불, 고객센터는 반복 공통 블록으로 템플릿화한다.
- 상품명, 옵션, 대표 이미지/상세 이미지 방향, 구매 이유, 사양/구성품은 Codex가 경쟁사 상세페이지와 Meta 광고 라이브러리 패턴을 분석해 먼저 제안한다.
- 가격은 해당 키워드의 네이버 쇼핑 상위 10위권 가격표를 만든 뒤 정한다. 브랜드 상품은 기본 배제하고, 브랜드 외 상품이 거의 없을 때만 브랜드를 포함한다. 최저가 1~2개를 그대로 따라가지 않고, 브랜드 제외 후 가격 오름차순 40~60% 구간의 중하 가격대를 기본 후보로 삼는다.
- 경쟁사 이미지는 구도/용도/톤 참고용이다. 원본 이미지, 로고, 문구, 브랜드 요소를 그대로 복제하지 않고 새 시안으로 만든다.
- 상세페이지에서 만든 대표/상세 이미지의 제품 톤, 색감, 착용 장면, 구매 이유가 Meta 광고 이미지의 기준이 된다. 광고 이미지는 타사 구도 참고를 하더라도 상세페이지와 다른 제품 세계관으로 만들지 않는다.
- Meta 광고 이미지 제작 전에는 타깃 키워드를 네이버 쇼핑에서 검색해 상위 10개 이미지 후보를 보고, 비브랜드 상품 중 소비자에게 가장 직관적인 이미지를 우선 참고한다. 비브랜드가 없을 때만 브랜드 이미지를 참고한다.
- Meta 광고 라이브러리에서는 정지 이미지 광고 소재를 우선 참고한다. 동영상 프레임이나 썸네일은 정지 이미지 후보가 없을 때 보조 참고로만 쓴다.

## 필요한 상품 입력값

| 항목 | 내용 |
|---|---|
| 상품명 | Codex가 경쟁사 표현을 참고해 초안 작성 |
| 카테고리 | 잇템몰 상품 목록 분류 |
| 가격 | 네이버 쇼핑 상위 10위권 가격표와 브랜드 제외 기준을 보고 중하 가격대로 초안 작성 |
| 옵션 | 색상, 사이즈, 타입, 수량 등 Codex가 초안 작성 |
| 사이즈 영향 여부 | 구매 판단/반품률에 영향이 있으면 사이즈 옵션, 사이즈 설명 이미지, 사이즈표 필수 |
| 대표 이미지 | 상품 카드와 상세 상단 이미지 시안 |
| 상세 이미지 | 사용컷, 구성컷, 기능 설명 이미지 시안 |
| 한 줄 카피 | 첫 화면에서 보이는 구매 이유 |
| 핵심 구매 이유 3개 | 상세페이지 중간 설득 블록 |
| 추천 대상 | 누구에게 맞는 상품인지 |
| 사양 | 소재, 크기, 무게, 구성품, 제조/수입 정보 초안 |
| 예외 정책 | 실제 공급/배송/인증 조건이 필요한 값은 사용자 확인 |

## 상세페이지 순서

## 아이템 전용 메인 페이지

- 키워드별 하위 URL은 해당 아이템 입장의 전용 메인 페이지로 본다.
- 예: `/tennis-bracelet/`는 테니스 팔찌 아이템의 전용 메인 페이지이며, 그 안에서 상품 CTA를 누르면 상세/주문 같은 더 깊은 하위 흐름으로 이동할 수 있다.
- 전용 메인 페이지 포인트 컬러는 Codex가 상품/타깃에 맞춰 먼저 제안한다. 사용자가 특정 색상을 지정하면 해당 아이템 전용 페이지 범위에서 우선 적용하고, 잇템몰 전체 공통 색은 함부로 바꾸지 않는다.

1. 상단 상품 구매 영역
   - 이미지 갤러리
   - 상품명
   - 한 줄 핵심 카피
   - 가격, 할인율, 배송 예상
   - 옵션 선택
   - 수량 선택
   - `바로 구매하기`, `N pay 구매`

2. 첫 설득 블록
   - 이 상품이 왜 지금 팔릴 만한지 설명
   - 핵심 구매 이유 3개를 짧고 명확하게 배치

3. 비주얼 상세 블록
   - 사용컷, 착용컷, 구성컷, 기능컷
   - 이미지 아래 문구는 1~2줄로 제한

4. 기능/장점 블록
   - 포인트 01
   - 포인트 02
   - 포인트 03

5. 추천 대상
   - `이런 분께 추천합니다`
   - 4개 내외로 작성

6. 상품 정보
   - 사이즈 영향 상품은 먼저 `사이즈 선택 가이드`를 넣는다.
   - 사이즈 설명 이미지. 이 이미지는 Codex가 SVG/HTML 도형으로 직접 그리지 말고 이미지 생성 도구로 만든다. 숫자, 표, 주의 문구는 깨짐을 막기 위해 HTML 텍스트와 표로 분리한다.
   - 사이즈표
   - 측정 기준, 권장 선택법, 실측 오차 안내
   - 제품명
   - 구성품
   - 소재
   - 크기/무게
   - 색상
   - 제조/수입 정보

7. 공통 안내 블록
   - 배송 안내
   - 교환/환불 안내
   - 고객센터

8. 하단 고정 CTA
   - `바로 구매하기`
   - `N pay 구매`
   - 모바일에서 항상 접근 가능하게 둔다.

## 공통 안내 블록 기본값

### 배송 안내

- 주문 접수 후 상품별 출고 일정을 안내한다.
- 배송 일정은 상품/입고 상태에 따라 달라질 수 있다.
- 제주/도서산간 지역은 추가 배송비 또는 배송 기간이 발생할 수 있다.

### 교환/환불 안내

- 상품 수령 후 사용 흔적, 구성품 누락, 포장 훼손 여부에 따라 교환/환불 가능 여부가 달라질 수 있다.
- 단순 변심 교환/환불은 상품별 정책과 왕복 배송비 기준을 따른다.
- 제품 하자 또는 오배송은 확인 후 교환/환불을 안내한다.

### 고객센터

- 운영시간: 평일 10:00 - 18:00
- 문의 채널: 잇템몰 고객센터 또는 주문 접수 연락처
- 문의 시 상품명, 주문자명, 연락처를 함께 남긴다.

## N pay 버튼 픽셀 기본값

- 버튼 라벨: `N pay 구매`
- 이벤트 의미: 구매 의향 클릭
- 전체 이벤트명: `NpayPurchaseClick`
- 상품별 이벤트명: `NpayPurchaseClick_{상품slug}`
- 상품별 버튼 속성: `data-pixel-event="NpayPurchaseClick_{상품slug}"`
- 운영 테스트 URL: `?pixel_test=1&test_run_id={날짜_상품slug_회차}`
- 확인 항목:
  - 상품 ID가 함께 기록된다.
  - 상품 slug가 함께 기록된다.
  - 상품명과 가격이 함께 기록된다.
  - 광고 UTM이 있으면 함께 기록된다.
  - 운영 서버가 `stored:true`로 응답한다.
  - 같은 `test_run_id` 테스트 로그를 삭제할 수 있다.
  - 테스트 클릭/삭제를 2회 반복해도 정상이다.
  - 상품이 여러 개일 때는 전체 이벤트가 아니라 상품별 이벤트 기준으로 N pay 클릭 여부를 판단한다.
  - 실제 네이버페이 결제 성공으로 해석하지 않는다.

## 3단계: 토스 PG 신청

토스 PG는 잇템몰 기준으로 신청한다. 목표는 잇템몰을 기존 사업자의 대표 PG 심사몰로 만들고, 승인 후 같은 사업자 기준의 추가 사이트/MID 적용 가능성을 검토하는 것이다.

대표님 준비물:

| 항목 | 상태 |
|---|---|
| 사업자등록증 사본 또는 사업자등록증명 | 준비 필요 |
| 통신판매업 신고번호/신고증 | 2단계 후 준비 |
| 구매안전서비스 이용확인증 | 1단계 후 준비 |
| 대표자 인증 | 대표님 진행 |
| 정산 계좌 | 대표님 확인 |
| 담당자 연락처/이메일 | 대표님 확인 |
| 잇템몰 URL | https://ittemmall.com |
| 판매 상품/가격/배송/환불 정책 | 사이트 반영 완료 |

Codex 처리:

1. 토스 심사 보완 요청이 오면 사이트 문구/정책/주문 흐름 수정
2. MID, client key, secret key 수령 후 설정 반영
3. public JS에는 client key만, secret key는 private config에만 분리
4. 테스트 결제/주문 저장소/주문조회 흐름 확인

토스 승인 후 받아야 할 값:

| 값 | 용도 | 보관 위치 |
|---|---|---|
| MID/상점아이디 | 상점 식별 | private config 및 운영 문서 |
| client key | 브라우저 결제창 초기화 | public config |
| secret key | 결제 승인 API | private config |
| variantKey | 결제위젯 구성 | public/private 설정 |

## 4단계: 운영 서버 private config

현재 `https://ittemmall.com/payment/healthcheck.php` 기준으로 PHP는 정상이나, private config와 주문 저장소는 아직 미설정이다.

필요한 서버 값:

| 항목 | 설명 |
|---|---|
| private config 경로 | public web root 밖 PHP 설정 파일 |
| 주문 저장 JSON 경로 | public web root 밖 주문 저장 파일 |
| 추적 로그 JSONL 경로 | public web root 밖 이벤트 로그 파일 |
| rate-limit JSON 경로 | public web root 밖 제한 상태 파일 |
| 관리자 토큰 | 주문 관리자/취소/알림 테스트 보호용 |

주의:

- 이 값들은 GitHub나 public_html에 올리지 않는다.
- Hostinger hPanel에서 public web root 밖에 폴더를 만들 수 있는지 확인해야 한다.
- 경로가 확인되면 Codex가 private config 생성 명령과 업로드 위치를 정리한다.

## 5단계: 네이버페이 결제형

네이버페이는 토스/통신판매업/정산 구조가 잡힌 뒤 진행한다. 결제형 기준으로 서류 심사 후 개발 인증정보가 나오고, 이후 개발/테스트/검수 순서로 진행한다.

대표님 준비물:

| 항목 | 상태 |
|---|---|
| 실명인증 완료된 네이버 아이디 | 대표님 확인 |
| 사업자등록증 사본 또는 사업자등록증명 | 준비 필요 |
| 통신판매업 신고번호/신고증 | 2단계 후 준비 |
| PG 거래금액 증빙 또는 매출 발생 내역 | 토스 승인/거래 후 준비 |
| PG 코드 월별 매출액 요약 이미지 | 네이버페이 요구 시 준비 |
| PG 코드 연동 URL 정보 이미지 | 네이버페이 요구 시 준비 |
| 담당자 이메일 | 대표님 확인 |
| 정산 계좌 | 대표님 확인 |

네이버페이 승인 후 받아야 할 값:

| 값 | 용도 | 보관 위치 |
|---|---|---|
| clientId | 네이버페이 public 설정 | public config |
| chainId | 네이버페이 가맹점 식별 | public/private 설정 |
| clientSecret | 승인/취소 API | private config |

Codex 처리:

1. 네이버페이 public config 생성
2. clientSecret은 private config에만 저장
3. return/approve/cancel URL 검수
4. 네이버페이 검수 체크리스트 대응
5. 운영 URL 결제 흐름 스모크 테스트

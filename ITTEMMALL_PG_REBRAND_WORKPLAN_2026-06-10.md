# 잇템몰 PG 대표몰 전환 작업 기록

작성일: 2026-06-10

## 결론

잇템몰은 기존 사업자의 통신판매업 신고 및 PG/간편결제 심사용 대표 쇼핑몰로 만든다.

현재 목표는 새 쇼핑몰을 처음부터 다시 기획하는 것이 아니라, 완성도가 높은 `마르세 스포츠` 구조를 잇템몰 기준으로 리브랜딩해 대체하는 것이다.

## 왜 이 작업을 하는가

- 잇템몰을 통신판매업 신고서에 기입할 대표 홈페이지로 세운다.
- 잇템몰 기준으로 토스 PG 심사를 준비한다.
- 토스 PG 승인 후 발급되는 상점아이디/MID와 결제 설정을 기준으로, 같은 사업자의 다른 홈페이지 결제 적용 가능성을 열어둔다.
- 네이버페이는 잇템몰 쇼핑몰 요건과 정책 페이지가 완성된 뒤 결제형 기준으로 신청 및 연동을 검토한다.
- 잇템몰은 기존 사업자용 대표몰이고, 신규 법인/별도 브랜드 작업과 섞지 않는다.

## 현재 이해한 사업 구조

| 구분 | 방향 |
|---|---|
| 잇템몰 | 기존 사업자 대표 종합몰 / PG 심사 허브 |
| 통신판매업 | 신고 홈페이지를 잇템몰 기준으로 정리 |
| 토스 PG | 잇템몰 기준으로 우선 신청 및 연동 준비 |
| 네이버페이 | 잇템몰 요건 완성 후 결제형 신청/연동 검토 |
| 다른 사이트 | 같은 사업자 기준으로 토스 PG 추가 사이트/MID/결제 UI 적용 가능성 검토 |

## 공식 조사 메모

### 토스페이먼츠

- 토스페이먼츠 공식 가이드에 따르면 상점아이디(MID) 추가 계약 신청이 가능하다.
- 결제수단 추가/변경 신청 시 사업자번호와 홈페이지 정보를 입력하고, 계약팀 검토 후 진행된다.
- 간편결제 추가/변경도 상점관리자의 MID 설정에서 변경 신청하는 흐름이 있다.
- 사업자가 다른 경우에는 MID 추가가 아니라 신규계약이 필요하다고 안내한다. 즉 이번 목적처럼 `같은 사업자` 기준의 대표몰을 먼저 세우는 것이 중요하다.
- 토스 개발자센터는 결제위젯에서 계약된 MID를 선택하고, 여러 결제 UI 또는 variantKey를 만들어 상황별 결제 UI를 쓸 수 있다고 설명한다.
- 토스 API 키는 전자결제 신청 이후 상점/MID 기준으로 확인하며, 클라이언트 키와 시크릿 키는 같은 세트를 써야 한다.

참고:

- https://www.tosspayments.com/blog/articles/35011?from=category
- https://docs.tosspayments.com/guides/v2/payment-widget/admin
- https://docs.tosspayments.com/reference/using-api/api-keys

### 네이버페이

- 네이버페이 공식 가이드는 결제형 가맹점 가입 조건, 취급 불가 상품, 심사 서류, 가입 신청 방법, 독립몰 연동 및 개발 검수 흐름을 별도로 안내한다.
- 네이버페이 결제형은 자체 개발 연동 및 테스트가 가능한 몰에 맞는 방향이다.
- 가입 완료 후에는 판매, 정산, 포인트 혜택, 가맹점 정보 관리 등을 네이버페이센터에서 운영한다.

참고:

- https://mkt.naver.com/naverpay_guide

## 작업 원칙

1. 잇템몰만 작업한다.
2. 마르세 스포츠는 구조 참조/이식 원본으로만 사용한다.
3. 사이트 표기, 헤더/푸터, SEO, canonical, tracking site payload, 결제 설정, 정책 페이지는 모두 `잇템몰 / ITTEMMALL / ittemmall.com` 기준으로 바꾼다.
4. 마르세 스포츠의 결제/주문/정책/운영점검 구조를 최대한 보존한다.
5. 잇템몰 안에 등록된 상품 브랜드가 `MARCE`라면 상품명, 상품 이미지, 상품 상세 카피의 MARCE 표기는 유지한다.
6. API 키, secret, 관리자 토큰, 실제 사업자 개인정보, 주문 데이터는 공개 저장소나 공개 웹루트에 넣지 않는다.
7. 로컬, GitHub, 운영 서버 반영 범위를 분리해서 보고한다.

| 원본 영역 | 잇템몰 적용 방향 |
|---|---|
| `index.html` | 잇템몰 대표 쇼핑몰 메인/상품/주문 흐름으로 리브랜딩 |
| `legal/` | 사업자정보, 이용약관, 개인정보처리방침, 환불/배송 정책을 잇템몰 기준으로 변환 |
| `payment/` | 주문 저장, 결제 준비, 네이버페이 준비, 관리자/조회 흐름을 잇템몰 기준으로 변환 |
| `payment/TOSS_SETUP.md` | Toss PG 우선 신청/키 발급/추가 사이트 전략을 오너 게이트로 분리 |
| `track.php` | 잇템몰 이벤트명/도메인 기준으로 유지 또는 변환 |
| `robots.txt`, `sitemap.xml` | 잇템몰 공개/비공개 경로 기준으로 재작성 |
| `DEPLOYMENT_MANIFEST.md`류 문서 | 잇템몰 PG 심사/배포 체크리스트로 새로 작성 |

## 작업 순서

1. 현재 잇템몰의 잘못된 초안 변경분을 확인한다.
2. 잇템몰 루트 작업본을 잇템몰 구조로 대체할 파일 범위를 정한다.
3. 마르세 스포츠 파일을 잇템몰 프로젝트로 복사한다.
4. 브랜드/도메인/SEO/정책/결제/추적 문구를 잇템몰 기준으로 치환한다.
5. 토스 PG 우선 적용을 전제로 결제 설정 placeholder와 private config 자리만 만든다.
6. 네이버페이는 기존 마르세 스포츠의 준비 구조를 잇템몰 기준으로 남겨두되, 실제 키는 오너 게이트로 분리한다.
7. 사업자정보 실제값 placeholder를 정책 페이지와 푸터에 일관되게 반영한다.
8. 로컬 브라우저에서 메인, 상품, 장바구니, 주문서, 결제 준비 페이지, 정책 페이지, 모바일 화면을 확인한다.
9. GitHub 반영 및 운영 배포는 별도 지시 또는 확인 후 진행한다.

## 주의

이번 작업은 피클볼 수요검증 기능을 새로 설계하는 작업이 아니다.

핵심은 `마르세 스포츠 완성 구조 -> 잇템몰 대표 심사몰` 전환이다. 필요한 경우 피클볼/다른 브랜드 결제 적용은 잇템몰 PG 승인 이후 같은 사업자 기준의 추가 사이트/MID/결제 UI 전략으로 별도 판단한다.

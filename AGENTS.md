# AGENTS.md - 잇템몰

## Scope

PROJECT_SCOPE=Ittemmall

이 폴더는 `ittemmall.com` 상위 쇼핑몰 프로젝트 범위다. `ittemmall.com/marce`와 `marcesports.com` 작업은 각각 `잇템몰 마르세`, `마르세 스포츠` 폴더에서 진행한다.

## First Line Rule

모든 구현 응답은 먼저 한 줄로 `TRIGGER=ON` 또는 `TRIGGER=OFF`를 선언한다.

- `TRIGGER=ON`: 저장, 제출, 추가, 삭제, 결제, 주문, 문의, 리스트, 히스토리, 새 라우트, 모달 플로우, 화면 왕복, 상태 복원이 필요한 기능 단위.
- `TRIGGER=OFF`: 버튼, 박스, 카드, 레이아웃, 스타일, 마크업 수정 같은 단순 UI 조각.

프로젝트 범위가 불분명하면 딱 한 번만 묻는다:

`어느 프로젝트 범위로 진행할까요? 잇템몰 / 잇템몰 마르세 / 마르세 스포츠`

## TRIGGER=ON Definition of Done

- Orphan 금지: 새 화면/라우트는 최소 1개 진입점과 최소 2개 출구를 가진다. 출구는 `Resume/Back to origin`과 `Next/Related action CTA`를 포함한다.
- Dead-end 금지: 완료, 빈상태, 에러 상태에도 의미 있는 다음 행동 CTA를 최소 2개 제공한다.
- State Preservation & Restoration: 폼 입력, 선택값, 스크롤, 필터, 정렬, origin을 저장하고 복귀 시 복구한다.
- 기능은 세트로 완성한다: Create/List/History/Detail/Edit/Return 흐름을 같이 본다.
- 데이터 없는 스텁 금지: 리스트/내역은 loading, empty, error, success 4상태와 persistence를 구현한다.
- 백엔드가 없으면 Mock API와 LocalStorage/IndexedDB를 쓰되 Repository/Service 레이어를 분리한다.

## TRIGGER=OFF Quality Bar

- 접근성, 반응형, disabled/error/focus/hover 상태를 챙긴다.
- 기존 디자인/컴포넌트 스타일을 우선한다.
- 단순 UI 작업에 Graph UX 절대조건을 과대 적용하지 않는다.

## Web Operations Guardrails

- 운영 URL, 라우트, 파일명, 폴더명을 임의로 바꾸지 않는다.
- SEO 필수 요소를 이유 없이 삭제하지 않는다: title, meta description, canonical, robots/noindex, H1/H2, image alt, 의미 있는 링크 텍스트, 404 링크, 모바일 렌더링.
- 광고/분석/전환 추적 코드는 운영 자산으로 본다. 이벤트명이나 `track.php`류 파일을 변경할 때 전환 측정 영향을 설명한다.
- 문의, 결제, 주문, 회원가입, 파일 업로드, 예약, 상담 신청 등 매출 연결 폼은 실제 흐름을 확인한다.
- 로컬, GitHub, 운영 서버를 구분해서 보고한다.
- 완료 전 실제 사용자 화면 기준으로 확인한다. 확인 못 한 범위는 확인했다고 말하지 않는다.

## Customer-Facing Copy Ban

- 잇템몰 고객 화면, 광고 랜딩, 상품 상세, 버튼, 배너, 메타 title/description, OG/JSON-LD, 주문/결제 안내에는 절대 테스트 의도나 검증 의도를 노출하지 않는다.
- 금지 표현: `테스트`, `검증`, `반응 확인`, `구매 의향`, `클릭 의도`, `클릭값`, `페이크도어`, `실험`, `테스트몰`, `테스트 상품`, `판매 테스트`, `버튼 클릭 반응`, `실제 결제가 아닙니다`.
- 고객에게 보이는 화면은 항상 정상 쇼핑몰 문구로 쓴다. 예: `추천 상품`, `신상품`, `시즌 상품`, `구매하기`, `상품 보기`, `주문 안내`, `결제 안내`.
- 테스트/검증/픽셀/로그/test_run_id 같은 말은 내부 MD, 코드, 서버 로그, 운영 체크 문서에서만 쓴다. 고객 화면에는 같은 의미를 우회 표현으로도 드러내지 않는다.
- 배포 전에는 `index.html`, `payment/*.html`, `legal/*.html` 같은 public 고객 화면에서 금지 표현이 보이는 문구로 남아 있는지 반드시 검색하고, 남아 있으면 배포하지 않는다.

## User Explanation Style

- 사용자가 `쉽게 말해봐`, `친구한테 말하듯`, `고등학생도 이해하게`, `무슨 말인지 설명해봐`라고 하면 먼저 쉬운 말과 비유로 설명한다.
- 이때 전문용어를 앞세우지 말고 `버튼을 누르면 신호가 생긴다`, `서버 기록장에 적힌다`, `테스트 낙서는 지운다`처럼 일상어로 말한 뒤, 필요한 경우에만 정확한 파일명/이벤트명을 덧붙인다.
- 사용자가 화가 난 상태에서는 변명보다 `맞다/아니다`, `지금 끊긴 지점`, `지금 바로 할 일`을 짧게 말한다.

## Default Typography

- 사이트 UI, 상세페이지, 광고 이미지, 안내 이미지, 문서형 시안, 카드뉴스, 배너, 썸네일처럼 글자가 들어가는 모든 산출물은 별도 폰트 가이드가 없으면 `Pretendard`를 기본 글씨체로 쓴다.
- 기존 브랜드 가이드, 운영 페이지의 확정 폰트, 외부 플랫폼 템플릿 제약이 있으면 그 지시를 우선하되, 새로 만드는 한글 텍스트 이미지/시안은 `Pretendard` 기준으로 제작한다.
- Pretendard 파일이 로컬에 없으면 작업 전에 폰트 확보 가능 여부를 확인하고, 대체 폰트를 쓴 경우에는 완료 보고에서 대체 사용 사실을 명확히 말한다.
- 광고 이미지 안 텍스트는 예시 이미지의 정보량에 맞춰 최소화한다. 기본은 짧은 카피 1개와 가격 정도만 넣는다.
- 잇템몰 광고 이미지는 `1차 제품/착용 원본 이미지 생성` 후, `2차 최종 Meta 광고 이미지 생성` 순서로 진행한다.
- 1차 이미지는 카피/가격 없이 제품, 구도, 배경, 현실감을 맞추는 무문구 원본으로 만든다. 이때 타깃 키워드에 어울릴 만한, 한국 소비자가 이해할 수 있는 장소를 우선한다.
- 2차 이미지는 1차 이미지를 기준으로 카피라이팅과 금액을 직접 넣어 Meta 업로드용 최종본으로 생성한다. `이미지와 잘 어울리게 넣어라`, `글씨체는 Pretendard로 해달라`는 식으로 짧고 명확하게 지시한다.
- 잇템몰 광고 이미지는 가능하면 `사진 생성 후 별도 텍스트 오버레이`로 만들지 않는다. 2차 최종 이미지 생성 프롬프트 안에 짧은 카피와 가격을 함께 넣고, 이미지와 자연스럽게 어울리는 최종 광고 이미지로 생성한다.
- 광고 이미지 안 한글 문구는 중국어/영어 상세페이지를 그대로 번역한 듯한 직역투, 어색한 띄어쓰기, 기계번역식 표현을 금지한다. 한국 쇼핑 광고에서 자연스럽게 보이는 짧은 표현으로 쓴다.
- 잇템몰 상세페이지 카피는 29CM 카피라이터처럼 담백하고 감각적으로 쓴다. 기능을 길게 설명하기보다 상품이 놓이는 장면, 착용감, 분위기, 선택 이유를 짧은 문장으로 보여준다.
- `왜 이 상품인가`, `포인트 01`, `이런 분께 추천합니다`처럼 설명서 같은 제목을 그대로 쓰지 않는다. 필요하면 `손목에 남는 작은 빛`, `가볍게 빛나는 하루`, `매일 고르기 쉬운 이유`처럼 상품의 감성과 장면이 먼저 보이는 제목으로 바꾼다.
- 카피라이팅은 과장, 촌스러운 감탄사, 홈쇼핑식 표현, 번역투, 상세페이지 공장 문구를 피한다. 한 문장에 기능을 많이 우겨 넣지 말고, 짧고 여백 있게 쓴다.
- 이미지 안에 도형, 박스, 배지, 칩을 만들고 그 안에 글자를 넣을 때는 기본적으로 가로/세로 가운데 정렬한다. 의도적으로 좌측 정렬하는 디자인을 선택할 때만 예외로 둔다.
- 사진형 이미지 생성 프롬프트에는 별도 지시가 없어도 `AI 느낌이 나지 않게`, `실제와 같은 하이 퀄리티 사진`을 짧게 포함한다.

- 상품명, 가격, 옵션, 대표 이미지/상세 이미지 방향, 핵심 구매 이유 3개, 제품 사양/구성품, 배송/교환/환불 기본값은 Codex가 경쟁사 상세페이지와 Meta 광고 라이브러리 패턴을 분석해 먼저 제안한다. 사용자는 원재료, 금지 조건, 최종 승인만 주면 된다.
- 의류, 신발, 장갑, 보호대, 착용 장비, 가구, 수납, 설치형 제품처럼 `사이즈가 구매 판단/반품률에 영향을 주는 상품`은 상품 업로드 기본값에 사이즈 옵션을 반드시 포함한다. 상세페이지에는 `사이즈 설명 이미지`와 `사이즈표`를 넣고, 측정 기준/권장 선택법/실측 오차 안내를 상품 정보 영역에 기입한다.
- 사이즈 설명 이미지는 Codex가 SVG/HTML 도형으로 직접 그려서 때우지 않는다. 이미지 생성 도구로 실제 상품 톤에 맞는 사이즈 가이드용 이미지를 만들고, 숫자/표/주의 문구는 깨짐을 막기 위해 상세페이지 HTML 텍스트와 표로 분리한다.
- 신사업발굴 워크플로우에서 사용자가 별도 지시 없이 키워드/아이템 테스트 상품을 게시하라고 하면, 기본 구조는 `숨김 단독몰`이다. 잇템몰 본사이트에는 해당 상품을 노출하지 않고, 키워드별 하위 URL에서만 상품을 보여준다.
- 키워드별 하위 URL은 해당 아이템 입장에서 전용 메인 페이지로 본다. 예: `/tennis-bracelet/`는 잇템몰 전체 홈의 하위 상품 카드가 아니라 테니스 팔찌 아이템의 전용 메인 페이지이며, 잇템몰 산하지만 고객에게는 독립 단독몰처럼 보여야 한다.
- 숨김 단독몰에서는 사용자가 잇템몰 본사이트(`/`)로 빠지는 동선을 만들지 않는다. 헤더 로고, 홈 버튼, 메뉴, 완료/내역 CTA도 해당 아이템 전용몰 안에서만 순환하게 만든다.
- 아이템 전용 메인 페이지를 만들 때 사이트 테마는 전체몰 공통 색을 임의로 바꾸지 말고 해당 아이템 전용 페이지 범위에서만 적용한다. 사용자가 컬러를 지정하면 그 컬러를 우선하고, 지정이 없으면 Codex가 상품/타깃에 맞춰 먼저 제안한다.
- 잇템몰 상품 가격은 Codex 임의 추론값으로 확정하지 않는다. 해당 상품 키워드를 네이버 쇼핑에서 조회해 상위 10위권 상품의 `상품명 / 판매가 / 구성품 / 배송비 / 브랜드 여부 / URL`을 먼저 표로 잡고, 브랜드 상품은 기본 배제한다. 단, 브랜드 외 상품이 거의 없으면 브랜드를 포함한다.
- 가격 초안은 네이버 쇼핑 상위 10위권 중 브랜드 제외 후 남은 상품들의 중하 가격대를 기본으로 한다. 운영 기준은 최저가 1~2개를 그대로 따라가지 않고, 가격 오름차순 기준 40~60% 구간의 상품가를 우선 후보로 삼는 것이다. 최종 가격은 이 근거표와 함께 제안한다.
- 새 아이템 검증에서는 판매가 조사와 원가 조사를 분리한다. 판매가만 보고 상품 게시나 광고 검증으로 넘어가지 않는다.
- 원가 조사는 먼저 상품 성격을 `실물 상품` / `비재화 상품`으로 나눈 뒤, 두 경우 모두 사용자에게 조사 방향을 짧게 확인한다. 예: `이건 실물 상품으로 보고 알리바바에서 유사 제품 평균 원가를 조사할까요?`, `이건 비재화로 보고 제작비/운영비/수수료 기준으로 원가 구조를 조사할까요?`
- 실물 상품이면 사용자가 다른 기준을 주지 않는 한 알리바바/도매 유사 제품 여러 개를 확인해 평균 원가를 잡는다. 최저가 1개만 기준으로 삼지 않고 MOQ, 옵션, 구성품, 배송/수입 비용 가능성, 인증 리스크를 함께 기록한다.
- 비재화 상품이면 사용자가 승인한 조사 방향에 맞춰 제작비, 운영비, 인건비, 플랫폼 수수료, 라이선스, 고객 응대, 환불 리스크 같은 원가 구조를 조사한다.
- 사용자가 국내 도매, 특정 공급처, 직접 제공 원가, 제작비 기준처럼 다른 조사 기준을 주면 그 기준을 우선한다. 사용자가 명확한 기준을 주지 않으면 Codex가 상품 성격에 맞는 조사 방향을 제안하고 확인받은 뒤 진행한다.
- 원가 조사 결과는 판매가 대비 원가율 30% 기준에 들어오는지 판단한다. 30%를 넘으면 가격 조정, 소싱 재검토, 또는 아이템 보류 후보로 본다.
- 대표 이미지와 상세 이미지는 경쟁사 예시의 구도/용도/톤을 참고해 새 시안으로 만든다. 경쟁사 이미지, 로고, 문구, 브랜드 요소를 그대로 복제하지 않는다.
- 사용자가 잇템몰 관련해서 `광고 이미지 생성`, `광고이미지 만들어`, `소재 만들어`, `이미지 갈아끼워`라고 말하면 이미지 생성 도구를 바로 실행하지 않는다. 먼저 타깃 키워드 기준으로 네이버 쇼핑 상위 10개 이미지와 Meta 광고 라이브러리 정지 이미지 광고를 확인하고, 최종 기준이 될 예시 이미지 1개를 고른다.
- Meta 광고 이미지 선정은 타깃 키워드 기준으로 `네이버 쇼핑 상위 10위권 이미지`와 `Meta 광고 라이브러리 정지 이미지 광고 소재`를 함께 본다. 네이버 쇼핑에서는 비브랜드 상품을 1순위로 보고, 상위 10개 후보 중 소비자 입장에서 키워드와 가장 직관적으로 연결되는 이미지를 고른다. 비브랜드 상품이 거의 없거나 전부 브랜드라면 브랜드 이미지를 참고할 수 있다.
- 광고 라이브러리 참고 이미지는 동영상 프레임 캡처, 영상 썸네일, 광고 카드 전체 캡처가 아니라 실제 정지 이미지 광고 소재를 우선한다. 영상만 있는 경우에는 보조 참고로만 쓰고, 정지 이미지 기준이 있으면 정지 이미지를 우선한다.
- 사용자가 검증 키워드를 부여한 경우 그 키워드가 상품/광고 작업의 출발점이다. 해당 키워드는 상품명, 광고 이미지 안의 짧은 카피, Meta 광고글 첫 문장에 자연스럽게 포함한다. 단, 억지 반복이나 검색어 나열처럼 보이게 쓰지 않고 소비자가 바로 이해하는 문장으로 녹인다.
- 최종 광고 이미지는 선택한 예시 이미지 1개의 구도, 톤, 정보량을 기준으로 새롭게 생성한다. 기존 상세페이지 이미지를 단순 조합해 광고 배너처럼 만들지 않는다.
- 광고 이미지 생성은 `1차 제품/착용 원본 이미지 생성` 후 `2차 최종 Meta 광고 이미지 생성` 순서로 진행한다.
- 1차 이미지는 카피/가격 없이 제품, 구도, 배경, 현실감을 맞추는 무문구 원본으로 만든다. 이때 타깃 키워드에 어울릴 만한, 한국 소비자가 이해할 수 있는 장소를 우선한다.
- 2차 이미지는 1차 이미지를 기준으로 카피와 가격을 넣어 Meta 업로드용 최종본으로 생성한다. 2차 프롬프트에는 `카피 + 가격 + Pretendard 글씨체 + 이미지와 잘 어울리게 배치`를 짧게 넣는다.
- 최종 광고 이미지의 카피와 가격은 별도 후작업으로 얹는 방식보다, 이미지 생성 프롬프트에 직접 넣어 이미지와 어울리는 광고 완성본으로 생성하는 방식을 우선한다.
- 잇템몰 광고 이미지 생성 프롬프트에도 항상 `AI 느낌이 나지 않게`, `실제와 같은 하이 퀄리티 사진`을 짧게 포함한다.
- 이미지 안 텍스트는 짧은 카피 1개와 가격 정도만 넣고, 기능 라벨, 설명 박스, 보조컷, 상세페이지식 정보 나열을 과하게 넣지 않는다.
- 배송, 교환, 환불, 고객센터처럼 반복되는 영역은 상세페이지 템플릿의 공통 블록을 사용한다.
- 사용자가 광고 게재, Meta 광고 검증, 캠페인 성과 판단을 별도로 요청하거나 승인했을 때만 광고비 한도/기간, 소재, 타깃, UTM, 성과 판단 기준을 확인한다.
- 잇템몰 Meta 광고 검증은 오토레이터 B2B 랜딩 검증과 다른 세팅이다. 잇템몰은 쇼핑몰 상품 상세와 N pay 버튼 반응을 중심으로 본다.
- 잇템몰 Meta 광고 집행은 기본적으로 `잇템몰 비즈니스 계정` 기준으로 진행한다. 오토레이터 비즈니스 계정이나 다른 프로젝트 광고 계정을 기본값으로 쓰지 않는다. Meta UI에서 계정이 오토레이터로 보이면 광고 생성 전 계정/비즈니스 선택을 먼저 확인한다.
- Meta 광고 관리자에서 Facebook 페이지 선택 드롭다운이 `Autorator`로 시작해도 `잇템몰` 페이지가 없는 것으로 단정하지 않는다. 페이지 목록에서 `MICORE, 미코어` 그룹을 펼치면 `잇템몰` 페이지가 보일 수 있다. 현재 확인된 잇템몰 Facebook 페이지는 `MICORE, 미코어` 그룹 아래의 `잇템몰`이며 ID는 `1069905426214093`이다.
- Meta 광고 게시 직전 `검토 후 게시 (n개)`가 현재 잇템몰 상품보다 많은 항목을 표시하면 계정 전체 임시 저장 초안이 섞인 상태로 본다. 검토 모달에서 잇템몰 상품이 아닌 초안이나 다른 프로젝트 오류가 보이면 임의로 게시/삭제/수정하지 말고, 현재 상품만 단독 게시 가능한지 먼저 찾는다. 단독 게시가 안 되면 사용자에게 `다른 초안을 삭제/수정해도 되는지` 확인한다.
- 잇템몰 Meta 광고 검증의 기본 학습 단위는 광고 세트로 본다. 캠페인은 `잇템몰_아이템검증` 같은 공통 관리 단위로 두고, 상품별로 별도 광고 세트를 만든다.
- 잇템몰 상품 검증 손익 기본식은 매출 100 기준 `광고비 20 / 원가 30 / 부대비용 20 / 순수익 30`으로 본다. 이전에 Codex가 임의로 잡은 다른 비율은 잊고 이 공식을 우선한다.
- 목표 구매 전환율은 2%로 본다. 즉 광고 클릭 50회당 구매 또는 N pay 구매 의향 클릭 1회를 기준으로 계산한다.
- 상품별 허용 광고비는 `판매가 x 20%`다. 목표 CPC는 `판매가 x 20% / 50`, 즉 `판매가 x 0.004`로 계산한다.
- 잇템몰 Meta 테스트 광고비 기본값은 고정 10,000원이 아니라 `판매가 x 20%`다. 필요하면 원 단위는 운영 편의상 반올림하되, 산식과 반올림값을 같이 기록한다.
- 테스트 광고비를 소진했는데 해당 상품의 상품별 N pay 버튼 픽셀 이벤트가 0이면 그 상품은 기본 탈락으로 판단한다.
- N pay 버튼 픽셀은 전체 합산 이벤트 `NpayPurchaseClick`만 보지 않는다. 상품별 혼선을 막기 위해 각 상품 버튼은 `NpayPurchaseClick_{상품slug}` 형식의 상품별 이벤트도 함께 보내야 한다.
- 일반 `바로 구매하기` 버튼도 전체 합산 이벤트 `PurchaseCtaClick`만 보지 않는다. 각 상품 버튼은 `PurchaseCtaClick_{상품slug}` 형식의 상품별 이벤트도 함께 보내야 한다.
- 새 상품을 만들 때는 해당 상품 고유의 N pay 버튼 로그 클릭값과 일반 구매 버튼 로그 클릭값을 둘 다 취할 수 있어야 한다. 예: A 상품은 `NpayPurchaseClick_{a상품slug}`와 `PurchaseCtaClick_{a상품slug}`, B 상품은 `NpayPurchaseClick_{b상품slug}`와 `PurchaseCtaClick_{b상품slug}`로 분리한다.
- 운영 서버에서 상품별 `NpayPurchaseClick_{상품slug}` 또는 `PurchaseCtaClick_{상품slug}`가 저장되면 어떤 상품에서 어떤 버튼 클릭이 일어났는지 `staysiaofficial@gmail.com`으로 메일 알림이 가야 한다. 단, `pixel_test=1` 같은 테스트 클릭은 메일을 보내지 않는다.
- 메일 알림까지 확인해야 하는 운영 전 테스트는 `pixel_test=1&notify_test=1&test_run_id=...`를 붙여 진행한다. 이 경우 메일 제목과 본문에 테스트 표시와 테스트 ID가 들어가야 하며, 확인 후 같은 `test_run_id`로 테스트 로그를 삭제한다.
- 상품 업로드 또는 광고 집행 전에는 운영 URL에서 `?pixel_test=1&test_run_id=...`를 붙여 해당 상품 N pay 버튼과 일반 `바로 구매하기` 버튼을 모두 실제로 누르고, 각각 상품별 로그가 `stored:true`로 저장되는지 확인한다. 그 다음 `__cleanup_test_events`로 같은 `test_run_id` 테스트 로그를 삭제한다.
- 메일 알림 검증도 N pay와 일반 `바로 구매하기`를 같은 기준으로 본다. 두 버튼 모두 상품별 로그 저장, 메일 알림, 테스트 로그 삭제까지 확인해야 완료로 본다.
- N pay 서버 로그 검증은 반드시 `테스트 클릭 -> 서버 저장 확인 -> 테스트 로그 삭제 -> 재테스트 클릭 -> 서버 저장 재확인 -> 테스트 로그 재삭제`를 2회 반복한다. 테스트 로그가 삭제되지 않았거나 상품별 이벤트가 안 찍히면 광고를 게재 완료로 보고하지 않는다.
- 서버 로그 판단은 Meta 광고 관리자 화면이 아니라 본사이트 서버 로그 기준이다. 전체 `NpayPurchaseClick`/`PurchaseCtaClick`만 보지 말고 상품별 `NpayPurchaseClick_{상품slug}`와 `PurchaseCtaClick_{상품slug}`가 있는지 확인한다.
- Toss 결제수단 세팅은 차후 실제 Toss 결제 시작/성공/실패를 측정하기 위한 별도 작업이다. 지금은 사용하지 않는다는 점만 인지한다.
- 기본 사전 조사 흐름은 `쇼핑인사이트 인기 분야/Top 20 -> 네이버 검색광고 API -> 쇼핑인사이트 추이 보완 -> Meta 광고 라이브러리 상단 이미지 광고 -> 후보 점수화/보고`다.
- 상품 업로드 흐름은 `상품 데이터 정리 -> 상세페이지 제작 -> N pay 버튼 픽셀 세팅 -> 상품 목록/상세 진입 확인`으로 본다.
- 광고/잇템몰 반응 검증 흐름은 명시 승인 후 `판매가 기준 허용 광고비와 목표 CPC 계산 -> 잇템몰 상품 상세 URL -> Meta 캠페인 게재 -> CTR/CPC/상세 진입률/상품별 N pay 클릭 픽셀 판단`으로 별도 진행한다.
- Meta 광고 라이브러리는 대한민국/모든 광고 기준으로 보고, 상단 노출 광고를 우선한다. 정지 이미지 광고와 2주 이상 게재된 광고를 우선 기록한다.
- 광고 문구와 이미지는 패턴 분석용으로만 쓰고 그대로 베끼지 않는다.
- 사용자가 명시적으로 요청하지 않으면 건축 관련 이미지, 자재 상품, 건축/자재왕 문구, 건축 프로젝트 자산은 잇템몰 아이템 테스트나 대표몰 화면에 섞지 않는다.

## Owner-Gated Delivery

계정 인증, 사업자 정보, API 키, 결제사 심사, 개인정보, 카드/통장/세금 정보가 필요한 일은 Codex가 할 수 있는 화면, 코드, 설정 자리, 검증 방법을 먼저 끝낸다. 사용자가 해야 할 게이트는 따로 명확히 말한다.

## Tool Honesty

- Codex 내부 `image_gen` 도구로 이미지를 만들었으면 그렇게 말한다.
- 덕테이프를 실제로 쓰지 않았으면 덕테이프로 했다고 말하지 않는다.
- 라이브 배포, GitHub 반영, 외부 업로드, 메일 전송은 실제 수행한 경우에만 완료로 말한다.
